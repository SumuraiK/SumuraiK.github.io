Git 的基本使用

Git的初始工作

在一个文件夹下mkdir name

  • 进行初始化

  • git add 项目或文档名 git add a.txt

  • git commit -m “你进行的操作的描述”

    commit只会提交在暂存区里面的东西,通过git add (name)将文件从工作区传入暂存区,通过commit将暂存区放到分支中去

  • git status

    当前的工作区的状态

  • git diff

    比较两次git之间的区别,可能会进行了更改

  • git log

    git log –pretty=oneline 精简版

    git log可以看到最近对文件进行的操作,上面在git commit -m “描述”

    那个描述就作为git log里面的操作名称

分支管理

git reset –hard HEAD^

​ > 用于reset,撤销操作,回到之前的操作

说明一下head的工作原理

其实head是处于git内部的一个指针,每一次操作都会有一个commit id

HEAD指针指向哪一个工作区,当前所在的工作区就是head所指的工作区

  • 来回到过去版本git reset --hard HEAD^
  • 当当前指针回到过去版本想要回到未来的版本,则需要commit id
  • 查看命令历史,HEAD指针的移动历史git reflog
  • 强制删除分支git branch -D <name>

基于head的分支使用

使用分支的目的是实现多人同时修改工程,但是互相不打扰对方的工作进程

Git鼓励大量使用分支：

  • 查看分支：git branch

  • 创建分支：git branch <name>

  • 切换分支：git checkout <name>

  • 创建+切换分支：git checkout -b <name>

  • 合并某分支到当前分支：git merge <name>

  • 删除分支：git branch -d <name>

  • 查看分支:git log --graph --pretty=oneline --abbrev-commit

其实创建分支就是修改HEAD的指向对象,新建分支,将head指向新的分支,则当前就工作在新的分支上,当新分支的工作完成后,就可以将master与新分支进行合并

Bug 分支

一般的master用来发布工程,工作人员在dev上工作,在工作时突然出现一个bug,新建bug分支进行修补

  • 保存当前工作状态 git stash
  • 转换到需要修补bug的分支(例子在master上)git checkout master
  • 在master上临时建立分支 git checkout -b issue-101
  • 修补完成后切换到master git checkout master
  • 合并并且删除 git merge --no-ff -m "merged bug fix 101" issue-101
  • 查看刚才保存的工作状态git stash list
  • 恢复stashgit stash apply
  • 删除stash list中的stash git stash drop
  • 恢复并且删除stashgit stash pop
  • 删除指定stash 1.git stash list 2.git stash apply stash@{0}

Github 和 Git的使用

  • 在Github创建一个仓库

下面的操作要在你想要push上去的仓库下

  • git remote add origin git@github.com:github的名字/learngit.git

  • git push -u origin master

  • 从github中克隆数据库git clone git@github.com:githubname/gitskills.git

    gitskills是你所想要克隆的项目的名字
